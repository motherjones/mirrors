new_member(child, index=None)¶

Add an existing Content object entry to the ordered list of members for this one. By default, this will simply append the child object to the list, but the user can specify the order if they so wish.

Elements inserted into the list at index n will shift all elements back by one.

Parameters:
  • child (Content) – the child Content object
  • index (integer) – (optional) the position within the list to put the child in
new_revision(data=None, metadata=None)¶

Create a new revision for this Content object. If the data is not in the correct format it will attempt to convert it into a bytes object.

Passing None for one of the arguments will result in that data not being changed.

Parameters:
  • data (bytes) – the actual content of the new revision
  • metadata (dict) – the new metadata
